컨버터의 경우, 인프라 로직에 가깝기 때문에 도메인 레이어보다는 인프라 레이어에 들어가는 것이 적합하다.

하지만, 그 경우 설계의 일관성을 챙기려면 JPA마저 인프라 레이어에 들어가게 되어 생산성이 크게 떨어진다.

- 도메인 레이어에서 @Convert 지정을 위해 해당 컨버터들을 참조하고 있다.
- 이를 인프라 레이어로 옮기면, 도메인 레이어가 인프라 레이어를 의존하는 현상이 발생한다.
- 이를 해결하려면 네 가지 방법을 고려해야 한다
    - JPA 엔티티를 인프라 레이어로 이동시키고, 도메인 레이어는 POJO로만 구성한다.
    - JPA를 도메인 레이어로 끌어올리고, 컨버터 또한 도메인 레이어에 위치시킨다
    - JPA가 인프라의 컨버터 레이어에 의존하는것을 허용시킨다.
    - 컨버터의 AutoApply를 True로 두어, 도메인 레이어에서 알 필요가 없게 만든다.

이 중 두번째 안을 선택하여 해당 컨버터 로직을 도메인 레이어에 두고, 도메인 레이어에서 Converter에 대한 의존성만 추가하는 형태로 구현하였다.
